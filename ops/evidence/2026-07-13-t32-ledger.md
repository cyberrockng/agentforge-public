# T3.2 Double-entry ledger + split engine

Date: 2026-07-13

## Scope implemented

- Added `@agentforge/payments` ledger primitives:
  - atomic USDT amounts,
  - split policies,
  - `ServiceCall` rows,
  - balanced `LedgerTransaction` entries,
  - JSONL journal persistence.
- Added split policies:
  - AgentForge-owned services: 100% Forge revenue, 0% founder payable.
  - Founder tenants: 80% founder payable, 20% Forge revenue.
- Wired runtime paid-call settlement to create:
  - a delivered `ServiceCall` row,
  - a balanced ledger transaction,
  - ledger IDs in the service response.
- Wired unpaid x402 quote responses to create a quoted `ServiceCall` row before returning the payment challenge.
  Superseded on 2026-07-14: anonymous unpaid quote responses no longer persist quoted rows to the
  durable journal; only settled paid calls are durable economic evidence.
- Backfilled the real First Heartbeat into the same accounting engine.

## First Heartbeat backfill

Backfill file: `ops/evidence/2026-07-13-t32-ledger-backfill.json`.

Important correction from AUDIT-2:

- The First Heartbeat tx was a paid call to `/svc/forge`.
- It generated the ShieldCheck package.
- It was not a paid call to `/svc/shieldcheck`.
- Therefore the backfill is accounted as an AgentForge-owned `forge` service call:
  - debit settlement wallet: `1000000` atomic USDT,
  - credit Forge revenue: `1000000` atomic USDT,
  - founder payable: `0`.

## Balanced entries invariant

The ledger rejects any transaction where debits and credits do not match per currency.

Test coverage:

- First Heartbeat backfill balances.
- Founder tenant split creates founder payable liability and Forge revenue.
- Split policies must sum to 10000 basis points.
- Malformed payment transaction refs are rejected.
- JSONL journal persists `ServiceCall` and `LedgerTransaction` rows.

## Runtime behavior

For callable paid tenants:

1. invalid body still returns `400` before payment verification,
2. unpaid valid body returns x402 payment challenge without writing anonymous quote rows to the durable journal,
3. verified and settled payment returns deliverable only after the delivered `ServiceCall` and balanced ledger transaction are written.

Journal path:

- `AGENTFORGE_LEDGER_PATH` if configured,
- otherwise `/data/agentforge/service-ledger.jsonl`.

Production note: before the next public paid call, mount a persistent Railway volume at `/data` or configure `AGENTFORGE_LEDGER_PATH` to another persistent external path. The code no longer relies on in-memory meters for accounting, but an unmounted container filesystem is not a long-term accounting database.
